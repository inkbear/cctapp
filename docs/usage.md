# General Usage

## Workflow

The usage of the CCT App entails the followng workflow:

Owner steps:

1. Create a CCT
1. Create a PI account
1. Assign the CCT to the PI

PI steps:

1. Create a project
1. Assign a CCT to the project

Optionally a Team may be created of collaborators and staff to assist with the folling steps:

1. Create a cohort
1. Add participants the cohort
1. Assign the cohort to a project
1. Create an experiment run from the project
    * Preview the experiment
    * Test the experiment
1. Run the experiment (send generated links to participants)
1. Review and export data

## Roles

There are 6 additive roles in an heierachical model where each role has all the capablities 
of the lower roles within a project context. 

1. Admin (set as a role)
1. Owner (set as a role)
1. PI (set as a role)
1. Collaborator (set via project assignment)
1. Staff (set via project assignment)
1. Unassigned Account (default level)

Any single user may concurrently hold the role 
of PI, collaborator, and/or staff depending on project assignment.

### Admin

Admin has access to everything including system configurations. This is a techincal role.

### Owner

Owner is the entity that owns the CCT and the CCT App. 

### PI

Primary Investigators are the single point of contact for use of CCT versions. They set up 
and run experiments using the CCT, projects and cohorts of participants with the aid of 
collaborators and staff.

### Collaborator

The Collaborator has nearly all the same powers as the PI but only for those projects 
the PI has assigned that person to. Collaborators can create cohorts, add particpants,
run experiments, and view data, all within the context of a project assignment.

### Staff

Staff are assigned to projects and can only manage participants, run experiments, 
and view data.

### Unassigned Account

Any person setting up an account from the sign up form is assigned the default role 
of staff but will not have any capablities until assigned to a project or elevated to the PI role.
